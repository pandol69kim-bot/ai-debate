## 발생 오류

```
pip : 'pip' 용어가 cmdlet, 함수, 스크립트 파일 또는 실행할 수 있는 프로그램 이름으로 인식되지 않습니다.
```

```
ERROR: Failed to build 'psycopg2-binary' when getting requirements to build wheel
pg_config executable not found.
```

---

## 원인 분석

### 1. `pip` 명령어 미인식
- 시스템 Python 3.14가 설치되어 있으나 `pip`가 PATH에 등록되지 않음
- 해결: `python -m pip` 형태로 사용

### 2. `psycopg2-binary` 빌드 실패
- Python 3.14는 최신 버전이라 `psycopg2-binary==2.9.10`의 pre-built wheel 미제공
- pip가 소스 빌드를 시도했으나 PostgreSQL 개발 도구(`pg_config`)가 없어 실패

### 3. `asyncpg`, `pydantic-core` wheel 부재
- 동일하게 Python 3.14용 pre-built wheel 없음 → 소스 빌드 필요
- 소스 빌드에는 C 컴파일러 및 Rust 툴체인 필요

---

## 해결 방법

### Step 1. `psycopg2-binary` 제거 (requirements.txt)
- `asyncpg`가 이미 PostgreSQL 비동기 드라이버로 포함되어 있어 중복
- `psycopg2-binary`는 동기 드라이버로 async FastAPI 프로젝트에서 불필요

**수정 파일:** `backend/requirements.txt`
```diff
- psycopg2-binary==2.9.10
```

### Step 2. Python 3.11 가상환경 생성
시스템에 Python 3.11이 설치되어 있음을 확인:
```
 -V:3.14 *        Python 3.14.3
 -V:3.11          Python 3.11 (64-bit)
 -V:Astral/CPython3.9.25 CPython 3.9.25 (64-bit)
```

Python 3.11로 가상환경 생성:
```powershell
cd backend
py -3.11 -m venv venv
```

### Step 3. 가상환경에 패키지 설치
```powershell
.\venv\Scripts\python.exe -m pip install -r requirements.txt
```

→ 모든 패키지 정상 설치 완료 ✅
