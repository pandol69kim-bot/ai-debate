# 작업계획: 관리자 메뉴 — 회원관리

- **날짜**: 2026-06-09
- **작업자**: Claude Code

---

## 목표

관리자 대시보드(`/admin`)에 회원관리 탭을 추가한다.
전체 회원 목록 조회, 관리자 권한 토글, 회원 삭제 기능을 제공한다.

---

## 구현 범위

### 백엔드

| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `/api/v1/admin/users` | GET | 전체 회원 목록 (이메일, 이름, is_admin, 토론수, 가입일) |
| `/api/v1/admin/users/{id}/toggle-admin` | PATCH | 관리자 권한 토글 |
| `/api/v1/admin/users/{id}` | DELETE | 회원 삭제 (본인 삭제 불가) |

### 프론트엔드

- `/admin` 페이지에 **토론 관리 / 회원 관리** 탭 추가
- 회원 목록 테이블: 이름, 이메일, 권한 배지, 토론수, 가입일, 액션 버튼
- 관리자 권한 토글 버튼 (즉시 반영)
- 회원 삭제 확인 모달

---

## 단계별 작업 목록

### Step 1. 백엔드 스키마 추가 (`schemas.py`)

```python
class AdminUserOut(BaseModel):
    id: UUID
    email: str
    name: str
    plan: str
    is_admin: bool
    debate_count: int = 0
    created_at: datetime
    model_config = {"from_attributes": True}
```

### Step 2. 백엔드 admin 라우터 확장 (`routes/admin.py`)

- `GET /admin/users` — 전체 유저 + 토론수 조회
- `PATCH /admin/users/{user_id}/toggle-admin` — `is_admin` 반전, 자기 자신 변경 금지
- `DELETE /admin/users/{user_id}` — 유저 삭제, 연관 Conversation.user_id NULL 처리 또는 cascade, 자기 자신 삭제 금지

### Step 3. 프론트엔드 API 클라이언트 (`client.ts`)

```typescript
export interface AdminUserOut {
  id: string;
  email: string;
  name: string;
  plan: string;
  is_admin: boolean;
  debate_count: number;
  created_at: string;
}

export const getAdminUsers = () =>
  api.get<AdminUserOut[]>("/admin/users");

export const toggleAdminUser = (id: string) =>
  api.patch<AdminUserOut>(`/admin/users/${id}/toggle-admin`);

export const deleteAdminUser = (id: string) =>
  api.delete(`/admin/users/${id}`);
```

### Step 4. 관리자 페이지 탭 구조 변경 (`app/admin/page.tsx`)

- 현재 단일 페이지 → 탭 컴포넌트로 분리
- `"토론 관리"` 탭: 기존 토론 목록 UI 유지
- `"회원 관리"` 탭: 신규 회원 테이블 UI

### Step 5. 회원 관리 UI 구현

- 회원 목록 테이블 (이름, 이메일, 권한 배지, 토론수, 가입일)
- 관리자 배지: `is_admin=true` 시 강조 표시
- 권한 토글 버튼: 클릭 즉시 낙관적 업데이트
- 삭제 확인 모달: 이름/이메일 표시 후 확인

---

## 주의사항

- 본인 계정(현재 로그인 관리자)은 권한 토글 및 삭제 불가 → 버튼 비활성화
- 회원 삭제 시 해당 유저의 토론(`Conversation.user_id`)을 NULL 처리하거나 함께 삭제할지 정책 결정 필요
  - 권장: `user_id = NULL` (토론 기록은 보존)
- 관리자가 1명뿐일 때 본인 관리자 권한 해제 방지 로직 고려
