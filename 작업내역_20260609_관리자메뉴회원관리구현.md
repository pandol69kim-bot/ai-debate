## 구현 상세

### 백엔드 API 엔드포인트

```
GET    /api/v1/admin/users                      — 전체 회원 목록 (토론수 포함)
PATCH  /api/v1/admin/users/{id}/toggle-admin    — 관리자 권한 토글
DELETE /api/v1/admin/users/{id}                 — 회원 삭제
```

**보호 로직**:
- 본인 계정은 권한 토글 및 삭제 불가 → 서버에서 400 반환
- 회원 삭제 시 연관 `Conversation.user_id`를 `NULL` 처리 (토론 기록 보존)

### 프론트엔드 관리자 페이지 탭 구조

기존 단일 페이지 → **토론 관리 / 회원 관리** 탭으로 분리

- **토론 관리 탭**: 기존 토론 목록 UI 유지
- **회원 관리 탭**: 신규 회원 테이블 UI

### 회원 관리 UI

- **회원 목록**: 권한 배지(관리자/일반), 이름, 이메일, 토론수, 가입일
- **관리자 토글**: `Shield` / `ShieldOff` 아이콘 버튼, 클릭 즉시 낙관적 업데이트
- **회원 삭제**: 확인 모달 (이름 + 이메일 표시) → 삭제 후 통계 카드 갱신
- **본인 보호**: 현재 로그인 계정의 토글·삭제 버튼 비활성화 + `(나)` 표시

### 공용 ConfirmModal 컴포넌트

토론 삭제 / 회원 삭제 모달을 `ConfirmModal` 컴포넌트로 통합
- `title`, `description`, `highlight`, `deleting`, `onCancel`, `onConfirm` props
