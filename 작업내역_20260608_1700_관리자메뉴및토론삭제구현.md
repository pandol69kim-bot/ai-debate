## 구현 상세

### 백엔드 API 엔드포인트

```
GET    /api/v1/admin/debates         — 전체 토론 목록 (user_email, round_count 포함)
GET    /api/v1/admin/stats           — 통계 (total_debates, done, failed, running, total_users)
DELETE /api/v1/admin/debates/{id}    — 토론 삭제 (관련 데이터 cascade 삭제)
```

**삭제 순서 (FK 제약)**:
1. `DebateRound` 삭제
2. `JudgeResult` 삭제
3. `ConsensusResult` 삭제
4. `Conversation` 삭제
5. `_debate_progress` 인메모리 딕셔너리 정리

**인증**: `get_admin_user` FastAPI Depends — JWT 토큰 검증 후 `is_admin=True` 확인, 미충족 시 403 반환.

### 프론트엔드 관리자 페이지 (`/admin`)

- **접근 제어**: `isLoggedIn && user?.is_admin` 체크, 미충족 시 `/`로 리디렉션
- **통계 카드**: 전체 토론, 완료, 진행 중, 실패, 가입 유저 (5개)
- **토론 목록**: 주제, 상태 배지, 모델 배지, 유저 이메일, 생성일, 삭제 버튼
- **삭제 확인 모달**: 삭제 전 주제 표시 후 확인 요청
- **낙관적 업데이트**: 삭제 성공 시 목록 및 통계 카드 즉시 반영

### 관리자 계정 현황

## 테스트 방법

1. `admin@arena.com`으로 로그인
2. 네비게이션 바에 "관리자" 메뉴 확인
3. `/admin` 페이지 접속 → 통계 카드 및 토론 목록 로딩 확인
4. 토론 항목 우측 휴지통 아이콘 클릭 → 삭제 확인 모달 → 삭제 실행
5. 일반 유저로 로그인 후 `/admin` 직접 접속 시 홈으로 리디렉션 확인
