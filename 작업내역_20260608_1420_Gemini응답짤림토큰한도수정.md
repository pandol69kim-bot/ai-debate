- **증상:** 토론에서 Gemini 의견이 문장 중간에 잘림

---

## 원인 분석

### 근본 원인: `max_output_tokens` 하드컷

```python
# debate_engine.py — 하드코딩된 800 토큰
adapter.generate(messages=[...], max_tokens=800)

# gemini_adapter.py — Gemini API에 직접 전달
generation_config = genai.types.GenerationConfig(max_output_tokens=max_tokens)
# → max_output_tokens=800 으로 Gemini에 전달됨
```

Gemini는 `max_output_tokens`를 **하드 컷**으로 처리 — 문장이 끝나지 않아도 800 토큰에서 강제로 응답을 자름.

### GPT/Claude와의 차이

| 모델 | max_tokens 처리 방식 | 짤림 여부 |
|------|---------------------|---------|
| GPT-4o | 소프트 가이드 (문장 완성 후 종료 경향) | 거의 없음 |
| Claude Opus | 소프트 가이드 | 거의 없음 |
| **Gemini** | **하드 컷 (토큰 수 도달 즉시 종료)** | **발생** |

### 추가 문제: config 값 미사용

`config.py`에 `MAX_TOKENS_PER_RESPONSE = 1000`이 정의되어 있었으나, `debate_engine.py`가 이를 사용하지 않고 `800`을 하드코딩.

---

## 수정 내용

### 수정된 파일 3곳

**1. `backend/app/engines/debate_engine.py`**
```python
# 수정 전 (2곳 모두)
adapter.generate(messages=[...], max_tokens=800)

# 수정 후
from app.core.config import settings
adapter.generate(messages=[...], max_tokens=settings.MAX_TOKENS_PER_RESPONSE)
```

**2. `backend/app/core/config.py`**
```python
# 수정 전
MAX_TOKENS_PER_RESPONSE: int = 1000

# 수정 후
MAX_TOKENS_PER_RESPONSE: int = 1500
```

**3. `.env`**
```
# 수정 전
MAX_TOKENS_PER_RESPONSE=1000

# 수정 후
MAX_TOKENS_PER_RESPONSE=1500
```
