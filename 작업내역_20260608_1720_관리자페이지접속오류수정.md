## 문제 현상

`admin@arena.com`으로 로그인 후 `http://localhost:3000/admin` 접속 시 홈(`/`)으로 리디렉션됨.

---

## 원인 분석

**`frontend/src/app/auth/login/page.tsx` 40번 줄**

로그인 성공 후 `getMe()` API로 유저 정보를 가져와 Zustand 스토어에 저장하는데,
`setAuth` 호출 시 `is_admin` 필드를 누락하여 전달했음.

```typescript
// 수정 전 — is_admin 누락
setAuth(access_token, { id: user.id, email: user.email, name: user.name, plan: user.plan });
```

결과적으로 Zustand에 저장된 `user.is_admin`이 항상 `undefined`(falsy)가 되어,
admin 페이지의 접근 제어 조건 `isLoggedIn && user?.is_admin`이 `false`로 평가 → 홈 리디렉션.

---

## 수정 내용

### 1. `frontend/src/lib/api/client.ts`

`getMe()` 반환 타입에 `is_admin: boolean` 추가.

```typescript
// 수정 전
export const getMe = () =>
  api.get<{ id: string; email: string; name: string; plan: string; created_at: string }>("/auth/me");

// 수정 후
export const getMe = () =>
  api.get<{ id: string; email: string; name: string; plan: string; is_admin: boolean; created_at: string }>("/auth/me");
```

### 2. `frontend/src/app/auth/login/page.tsx`

`setAuth` 호출에 `is_admin` 필드 추가.

```typescript
// 수정 전
setAuth(access_token, { id: user.id, email: user.email, name: user.name, plan: user.plan });

// 수정 후
setAuth(access_token, { id: user.id, email: user.email, name: user.name, plan: user.plan, is_admin: user.is_admin ?? false });
```

## 적용 방법

이미 로그인된 상태라면 **로그아웃 후 재로그인** 필요.
(Zustand persist 미들웨어가 localStorage에 캐싱한 이전 상태에는 `is_admin`이 없음)
