# 작업 계획: 인증 유저만 토론 가능하도록 제한

## 목표
초기 화면(`/`)에서 로그인 여부를 확인하여 비인증 유저는 토론을 시작할 수 없도록 처리.
프론트엔드 UI + 백엔드 API 양쪽에 인증 가드 적용.

---

## 현황 파악

### 현재 초기 화면 (`frontend/src/app/page.tsx`)
- `useAuthStore` 미사용 — 인증 체크 없음
- 누구나 토론 주제 입력 후 "토론 시작" 버튼 클릭 가능
- `startDebate()` API 호출 시 JWT 토큰 없어도 서버가 수락

### 현재 백엔드 (`backend/app/api/routes/debate.py`)
- `POST /debate/start` 엔드포인트에 인증 의존성(Depends) 없음
- API 직접 호출로도 토론 시작 가능 → 보안 취약

---

## 적용할 UX 방식: **소프트 블록 (Soft Block)**

> 하드 리다이렉트(페이지 즉시 이동)보다 소프트 블록이 UX상 낫다.
> 비로그인 유저도 UI를 탐색하고 주제 예시를 볼 수 있되,
> 토론 시작 시점에 로그인을 유도한다.

### 비로그인 시 화면
- 폼(주제 입력, 모델 선택, 라운드 선택) → 그대로 표시 (탐색 허용)
- CTA 버튼 → "로그인하고 토론 시작" 버튼으로 교체 (accent 스타일 유지)
- 폼 상단에 안내 배너 추가: "토론에 참여하려면 로그인이 필요합니다"

### 로그인 시 화면
- 기존과 동일 — 정상 토론 시작 가능

## 변경 파일 목록

### 1. `frontend/src/app/page.tsx` ★ 핵심

**변경 내용:**
```
- useAuthStore import 추가
- isLoggedIn 상태 구독
- CTA 버튼 분기 처리:
    isLoggedIn → 기존 "토론 시작" 버튼
    !isLoggedIn → "로그인하고 토론 시작" 버튼 (router.push("/auth/login"))
- 비로그인 시 안내 배너 추가 (폼 상단)
- handleStart()에 isLoggedIn 체크 추가 (직접 호출 방어)
```

**추가할 코드 위치:**
- import: `useAuthStore` from `@/lib/store/authStore`
- `const { isLoggedIn } = useAuthStore();` — 기존 state 선언 다음
- CTA 버튼 교체: 기존 `<button onClick={handleStart}>` 블록을 조건부 렌더링으로 교체
- 배너: 폼 영역(`<div className="flex-1 ...">`) 최상단에 삽입

### 2. `backend/app/api/routes/debate.py` ★ 보안

**변경 내용:**
```
- POST /debate/start 엔드포인트에 JWT 인증 의존성 추가
- 인증 실패 시 401 Unauthorized 반환
- 인증된 user 정보를 conversation 생성 시 user_id에 연결
```

**추가할 코드:**
```python
from app.api.routes.auth import _get_user_from_token
from fastapi import Header

@router.post("/start", response_model=...)
async def start_debate(
    payload: DebateStartRequest,
    db: AsyncSession = Depends(get_db),
    authorization: Optional[str] = Header(default=None),
):
    user = await _get_user_from_token(authorization, db)
    # user.id → conversation.user_id 저장
```

> 또는 별도 `get_current_user` 의존성 함수로 분리 (재사용성 고려)

### 3. `backend/app/api/routes/auth.py` (선택적 리팩토링)

현재 `_get_user_from_token()`이 `auth.py` 내부 private 함수.
다른 라우터에서 재사용하기 위해 `deps.py` 또는 `auth.py`에서 export 가능하도록 정리.

## 구현 순서

```
Step 1. 백엔드 - 인증 의존성 함수 정리 (auth.py or deps.py)
Step 2. 백엔드 - /debate/start에 인증 guard 적용
Step 3. 프론트엔드 - page.tsx에 useAuthStore 추가
Step 4. 프론트엔드 - 비로그인 배너 UI 추가
Step 5. 프론트엔드 - CTA 버튼 조건부 분기 처리
Step 6. 동작 확인:
    - 비로그인 → UI 배너 + 버튼 변경 확인
    - 비로그인 → API 직접 호출 시 401 반환 확인
    - 로그인 후 → 정상 토론 시작 확인
```

## 예외 처리

| 상황 | 처리 방식 |
|------|----------|
| 비로그인 상태에서 handleStart() 직접 호출 | router.push("/auth/login") |
| 로그인 후 토큰 만료 (401 응답) | 에러 메시지 "세션이 만료되었습니다. 다시 로그인해주세요." + logout() 호출 |
| 백엔드 API 401 수신 | axios 인터셉터 또는 handleStart catch에서 처리 |
