# 제미나이 토론 답변 짤림 원인 파악

## 조사 일시
2026-06-08 15:20

## 증상
- 토론 페이지에서 Gemini 2.5 답변이 문장 중간에 잘림
- 예: "핵심적인 중견국(Middle Power)으로서의 위상을 더욱 공" → 이후 내용 없음
- 다른 모델(GPT, Claude)은 정상 표시됨

---

## 조사 범위

### 1. 프론트엔드 CSS 조사 → 문제 없음

**파일**: `frontend/src/app/globals.css`

`.prose-arena` 클래스 정의:
```css
.prose-arena {
  @apply text-slate-200 leading-relaxed;
}
.prose-arena p { @apply mb-3; }
.prose-arena ul, .prose-arena ol { @apply pl-5 mb-3 space-y-1; }
/* ... */
```

- `max-height` → 없음
- `overflow: hidden` → 없음
- `-webkit-line-clamp` → 없음
- `height` 제한 → 없음

**결론**: CSS 표시 문제 아님. 컴포넌트(`ModelCard.tsx`, `DebateTimeline.tsx`)에도 height 제약 없음.

---

### 2. 백엔드 토큰 한도 조사 → 원인 확인

**파일**: `backend/app/engines/debate_engine.py`

이전 코드 (버그):
```python
adapter.generate(
    messages=[...],
    max_tokens=800,  # 하드코딩
)
```

수정 후 코드:
```python
adapter.generate(
    messages=[...],
    max_tokens=settings.MAX_TOKENS_PER_RESPONSE,  # 설정값 사용
)
```

**파일**: `backend/app/core/config.py`
```python
MAX_TOKENS_PER_RESPONSE: int = 1500  # 800에서 수정됨
```

**파일**: `backend/app/adapters/gemini_adapter.py`
```python
generation_config = genai.types.GenerationConfig(max_output_tokens=max_tokens)
```

## 핵심 원인 (3가지)

### 원인 1: max_tokens=800 하드코딩 (주 원인)
- `debate_engine.py`에 `max_tokens=800`이 하드코딩
- Gemini는 `max_output_tokens`를 **엄격한 상한**으로 처리 → 토큰 소진 시 문장 중간에 강제 종료
- GPT, Claude는 응답이 자연스럽게 마무리되는 경향 → 같은 설정에도 상대적으로 덜 잘림
- **→ 이전 세션에서 `settings.MAX_TOKENS_PER_RESPONSE` 사용으로 이미 수정됨**

### 원인 2: DB에 잘린 데이터가 저장됨 (기존 토론 영구 손상)
- 800 토큰 시절 생성된 토론 데이터가 DB에 truncated 상태로 저장
- REST API로 DB에서 불러오는 기존 토론은 **복구 불가**
- 스크린샷에 보인 잘린 답변 = 이 케이스

### 원인 3: 1500 토큰도 한국어 기준 부족할 수 있음
- 한국어는 토크나이저 기준 1글자 = 약 2~4 토큰
- 1500 토큰 ≈ 한국어 400~750자 분량
- 심화 토론 주제(지정학, 경제정책 등)의 충분한 논거 제시에는 3000 토큰 이상 권장
- **→ 3000으로 상향 예정**

## 조치 현황

| 항목 | 상태 |
|------|------|
| `debate_engine.py` max_tokens 하드코딩 제거 | 완료 (이전 세션) |
| `config.py` MAX_TOKENS_PER_RESPONSE 1500으로 상향 | 완료 (이전 세션) |
| `.env` MAX_TOKENS_PER_RESPONSE=1500 설정 | 완료 (이전 세션) |
| 3000으로 추가 상향 | 미적용 (사용자 확인 후 진행 예정) |
| 기존 잘린 토론 데이터 복구 | **불가** — 새 토론 재실행 필요 |
| uvicorn 서버 재시작 | **필요** — 설정 변경사항 반영 위해 |

---

## 해결을 위해 필요한 작업

1. **MAX_TOKENS_PER_RESPONSE를 3000으로 상향**
   - `config.py`: `MAX_TOKENS_PER_RESPONSE: int = 3000`
   - `.env`: `MAX_TOKENS_PER_RESPONSE=3000`

2. **uvicorn 백엔드 서버 재시작**
   ```bash
   # start-dev.bat 재실행 또는
   cd backend
   uvicorn app.main:app --reload --port 8000
   ```

3. **기존 잘린 토론은 동일 주제로 새 토론 재실행**
   - DB에 저장된 truncated content는 복구 불가
   - 새 토론 실행 시 3000 토큰 한도로 완전한 답변 저장됨

## 모델별 max_output_tokens 최대 한도 (참고)

| 모델 | 최대 출력 토큰 |
|------|--------------|
| gemini-2.5-flash | 8,192 |
| gpt-4o | 4,096 (기본) / 16,384 (max) |
| claude-opus-4-8 | 32,000 |

3000 토큰은 세 모델 모두 지원 가능한 안전한 값.
