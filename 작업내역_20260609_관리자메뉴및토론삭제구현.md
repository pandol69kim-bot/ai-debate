- **작업자**: Claude Code

## 작업 개요

관리자 전용 메뉴 및 토론 삭제 기능을 구현했습니다.
로그인한 유저 중 `is_admin=True` 계정만 접근 가능한 관리자 대시보드를 추가했습니다.

## 구현 상세

### 백엔드 API 엔드포인트

```
GET    /api/v1/admin/debates         — 전체 토론 목록 (user_email, round_count 포함)
GET    /api/v1/admin/stats           — 통계 (total_debates, done, failed, running, total_users)
DELETE /api/v1/admin/debates/{id}    — 토론 삭제 (관련 데이터 cascade 삭제)
```

**삭제 순서 (FK 제약)**:
1. `DebateRound` 삭제
2. `JudgeResult` 삭제
3. `ConsensusResult` 삭제
4. `Conversation` 삭제

**인증**: `get_admin_user` FastAPI Depends — JWT 검증 후 `is_admin=True` 확인, 미충족 시 403 반환

### 프론트엔드 관리자 페이지 (`/admin`)

- **접근 제어**: `isLoggedIn && user?.is_admin` 체크, 미충족 시 `/`로 리디렉션
- **통계 카드**: 전체 토론, 완료, 진행 중, 실패, 가입 유저 (5개)
- **토론 목록**: 주제, 상태 배지, 모델 배지, 유저 이메일, 생성일, 삭제 버튼
- **삭제 확인 모달**: 삭제 전 주제 표시 후 확인 요청
- **낙관적 업데이트**: 삭제 성공 시 목록 및 통계 카드 즉시 반영

### NavBar 모바일 반응형

- `md(768px)` 기준 데스크톱/모바일 분기
- 모바일: 햄버거(☰) / 닫기(✕) 버튼 + 드롭다운 메뉴
- `pathname` 변경 시 드롭다운 자동 닫힘

### 관리자 로그인 오류 수정

로그인 시 `setAuth`에 `is_admin` 필드 누락으로 관리자 페이지 리디렉션되던 문제 수정
- `client.ts` `getMe()` 반환 타입에 `is_admin: boolean` 추가
- `login/page.tsx` `setAuth` 호출에 `is_admin: user.is_admin ?? false` 추가
