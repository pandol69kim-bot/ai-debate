## 상세 변경 내용

### 1. `client.ts` — Auth API + 인터셉터

```typescript
// JWT 자동 첨부 인터셉터
api.interceptors.request.use((config) => {
  const token = localStorage.getItem("arena_token");
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// 추가 함수
register(email, name, password)   // POST /api/v1/auth/register
login(email, password)            // POST /api/v1/auth/login
getMe()                           // GET  /api/v1/auth/me
```

### 2. `authStore.ts` — Zustand 인증 스토어

```typescript
// 상태
{ token, user: { id, email, name, plan }, isLoggedIn }

// persist 미들웨어로 localStorage에 자동 저장/복원
// 브라우저 새로고침 후에도 로그인 상태 유지

setAuth(token, user)  // 로그인 후 호출
logout()              // 토큰 삭제 + 상태 초기화
```

### 3. `/auth/register` — 회원가입 페이지

- 필드: 이메일, 이름, 비밀번호, 비밀번호 확인
- 클라이언트 유효성 검사:
  - 이메일 형식 검사
  - 이름 2자 이상
  - 비밀번호 8자 이상
  - 비밀번호 일치 여부
- 회원가입 성공 → 자동 로그인 → 홈 이동
- 중복 이메일: "이미 사용 중인 이메일입니다." 에러 표시
- 비밀번호 표시/숨김 토글

### 4. `/auth/login` — 로그인 페이지

- 필드: 이메일, 비밀번호
- 로그인 성공 → `/auth/me` 호출로 사용자 정보 취득 → 홈 이동
- 잘못된 인증정보: "이메일 또는 비밀번호가 올바르지 않습니다." 에러 표시
- 비밀번호 표시/숨김 토글

### 5. `NavBar.tsx` — 내비게이션 업데이트

**로그아웃 상태:**
```
[Arena] [Rankings]  |  [로그인] [회원가입]
```

**로그인 상태:**
```
[Arena] [Rankings]  |  [👤 홍길동] [로그아웃]
```

### 6. `backend/app/api/routes/auth.py` — `/auth/me` 엔드포인트 추가

```python
GET /api/v1/auth/me
Authorization: Bearer {token}

# → UserOut (id, email, name, plan, created_at)
```

JWT 토큰에서 user_id를 추출하여 DB에서 사용자 정보를 조회하여 반환.

## 인증 흐름

### 회원가입 플로우
```
사용자 입력 → 유효성 검사 → POST /auth/register
  → 성공: POST /auth/login → setAuth(token, user) → / 이동
  → 실패: 에러 메시지 표시
```

### 로그인 플로우
```
사용자 입력 → POST /auth/login → token 획득
  → GET /auth/me (Bearer token) → user 정보 취득
  → setAuth(token, user) → / 이동
```

### 세션 유지
- Zustand `persist` 미들웨어로 `localStorage["arena_auth"]`에 저장
- 페이지 새로고침 후에도 로그인 상태 자동 복원
- `localStorage["arena_token"]`은 axios 인터셉터가 자동 첨부

### 로그아웃
```
logout() → localStorage 삭제 → Zustand 초기화 → / 이동
```

## 기술 주의사항

### bcrypt 응답 지연
로그인 시 bcrypt 해싱으로 2~5초 소요. axios 타임아웃 30초로 설정됨 (문제없음).

### SSR 안전성
- `localStorage` 접근 시 `typeof window !== "undefined"` 가드 적용
- Zustand persist는 hydration 이후에만 동작
