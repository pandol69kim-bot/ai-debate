## 발생 오류

```
socket.gaierror: [Errno 11001] getaddrinfo failed
ERROR: Application startup failed. Exiting.
```

**발생 위치:** `backend/app/main.py` lifespan → `engine.begin()` (DB 최초 연결)

---

## 원인 분석 (3가지 복합)

### 1. Docker 포트 미노출
`docker-compose.yml`에서 postgres/redis 서비스에 `ports` 설정이 없어  
Docker 내부 네트워크에서만 접근 가능, 호스트(로컬 uvicorn)에서는 접근 불가.

```
# 기존 (포트 노출 없음)
ai-postgres-1   5432/tcp       ← 호스트 접근 불가
ai-redis-1      6379/tcp       ← 호스트 접근 불가

# 수정 후
ai-postgres-1   0.0.0.0:5432->5432/tcp   ← 호스트 접근 가능
ai-redis-1      0.0.0.0:6379->6379/tcp   ← 호스트 접근 가능
```

### 2. config.py 기본값이 Docker 내부 서비스명
```python
# 기존 (Docker 내부에서만 동작)
DATABASE_URL = "postgresql+asyncpg://arena:arena_pass@postgres:5432/arena_db"
REDIS_URL = "redis://redis:6379/0"

# 수정 후 (로컬 개발 기본값)
DATABASE_URL = "postgresql+asyncpg://arena:arena_pass@localhost:5432/arena_db"
REDIS_URL = "redis://localhost:6379/0"
```

### 3. .env 파일 로드 실패
`config.py`의 `env_file = ".env"` 설정은 uvicorn 실행 위치 기준으로 `.env`를 탐색.  
`backend/` 폴더에서 실행 시 `backend/.env`를 찾지만, 실제 `.env`는 프로젝트 루트에 존재.  
→ `.env` 로드 실패 → 기본값(`postgres`) 그대로 사용 → getaddrinfo 실패

---

## 해결 방법

### Step 1. docker-compose.yml에 포트 노출 추가

```yaml
# postgres 서비스에 추가
ports:
  - "5432:5432"

# redis 서비스에 추가
ports:
  - "6379:6379"
```

컨테이너 재시작:
```powershell
docker-compose up -d postgres redis
# → Container ai-postgres-1 Recreated
# → Container ai-redis-1 Recreated
```

### Step 2. .env 호스트명 수정 (로컬 개발용)

```diff
- DATABASE_URL=postgresql+asyncpg://arena:arena_pass@postgres:5432/arena_db
- DATABASE_URL_SYNC=postgresql://arena:arena_pass@postgres:5432/arena_db
- REDIS_URL=redis://redis:6379/0

+ DATABASE_URL=postgresql+asyncpg://arena:arena_pass@localhost:5432/arena_db
+ DATABASE_URL_SYNC=postgresql://arena:arena_pass@localhost:5432/arena_db
+ REDIS_URL=redis://localhost:6379/0
```

### Step 3. config.py 기본값 및 env_file 경로 수정

```python
# 기본값 localhost로 변경
DATABASE_URL: str = "postgresql+asyncpg://arena:arena_pass@localhost:5432/arena_db"
REDIS_URL: str = "redis://localhost:6379/0"

# env_file 다중 경로 설정 (backend/ 또는 루트 어디서 실행해도 .env 로드)
class Config:
    env_file = (".env", "../.env")
    extra = "ignore"
```

## 변경된 파일

| 파일 | 변경 내용 |
|------|-----------|
| `docker-compose.yml` | postgres/redis에 `ports` 섹션 추가 |
| `.env` | DB/Redis URL 호스트명 `postgres`/`redis` → `localhost` |
| `backend/app/core/config.py` | 기본값 `localhost`로 변경, `env_file` 다중 경로 설정 |

---

## 로컬 개발 vs Docker 배포 URL 비교

| 환경 | DATABASE_URL 호스트 | 설정 위치 |
|------|---------------------|-----------|
| 로컬 개발 (uvicorn) | `localhost` | `config.py` 기본값 또는 `.env` |
| Docker 전체 실행 | `postgres` | `docker-compose.yml` environment 오버라이드 |

> Docker의 backend 서비스는 `environment`에서 `DATABASE_URL`을 `postgres`로 오버라이드하므로  
> `config.py` 기본값이 `localhost`여도 Docker 환경에서는 정상 동작.
