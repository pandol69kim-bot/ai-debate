## 1. 프로그램 소개

**AI Arena**는 여러 AI 모델(GPT-4o, Claude Opus, Gemini 2.5)에 동일한 질문을 던지고, AI들이 서로 토론·반박한 뒤 Judge AI가 최종 승자를 선정하는 플랫폼입니다.

### 핵심 기능

| 기능 | 설명 |
|------|------|
| **회원 인증** | 회원가입/로그인으로 개인 토론 기록 관리 |
| **멀티 AI 동시 질의** | 선택한 AI들이 동일한 질문에 병렬로 답변 |
| **AI 토론 (최대 3라운드)** | 라운드마다 상대 답변을 보고 반박·보완 |
| **Judge AI 평가** | 정확성·논리성·근거·창의성·실현가능성 5항목 채점 |
| **합의 결론 도출** | AI들의 공통 의견을 종합한 최종 결론 생성 |
| **토론 기록 조회** | 과거 토론 목록 및 전체 내용 재열람 |
| **ELO 랭킹** | 토론 결과가 누적되어 AI 모델 순위 형성 |
| **실시간 스트리밍** | 토론 진행 상황을 실시간으로 화면에 출력 |

> **토론 참여 조건**: 로그인한 회원만 토론을 시작할 수 있습니다. 비회원은 화면 탐색은 가능하나 토론 시작이 제한됩니다.

## 4. 실행 방법

### 방법 A — 원클릭 실행 (권장)

`start.bat` 파일을 **더블클릭**합니다.

```
처음 실행 시 이미지 빌드로 5~10분 소요
이후 실행은 30초 내외
```

완료 후 브라우저에서 접속:
- **서비스**: http://localhost
- **API 문서**: http://localhost/docs

---

### 방법 B — 개발 모드 (코드 수정 시)

**1단계**: `start-dev.bat` 실행 → PostgreSQL, Redis 컨테이너 시작

**2단계**: Python 가상환경 및 백엔드 실행 (새 터미널)

```cmd
cd "D:\_웹어플\AI 멀티 모델 경쟁·토론 플랫폼\backend"

:: 최초 1회: 가상환경 생성
python -m venv venv

:: 가상환경 활성화
venv\Scripts\activate

:: 패키지 설치
pip install -r requirements.txt

:: 서버 실행
uvicorn app.main:app --reload --port 8000
```

> Python 3.11 사용을 권장합니다. 3.12/3.13 이상은 일부 패키지(pywin32 등) 호환 문제 발생 가능.

**3단계**: 프론트엔드 실행 (새 터미널)

```cmd
cd "D:\_웹어플\AI 멀티 모델 경쟁·토론 플랫폼\frontend"
npm install
npm run dev
```

접속 주소:
- **프론트엔드**: http://localhost:3000
- **백엔드 API**: http://localhost:8000
- **API 문서 (Swagger)**: http://localhost:8000/docs

## 5. 화면별 사용법

### 화면 1 — 홈 (토론 시작)

접속 주소: `http://localhost`

#### 비로그인 상태

- 폼(주제·모델·라운드 선택) 탐색 가능
- 토론 시작 버튼 → "로그인하고 토론 시작" 으로 표시
- 클릭 시 로그인 페이지로 이동

#### 로그인 상태 — 토론 시작하기

**① 토론 주제 입력**
- 텍스트 박스에 토론할 주제나 질문을 입력합니다.
- 하단의 예시 버튼을 클릭하면 주제가 자동 입력됩니다.

```
예시 주제:
- 2030년 가장 유망한 투자 자산은 무엇인가?
- AI가 인간의 일자리를 대체할 것인가?
- 원격 근무와 사무실 근무 중 어느 것이 더 효율적인가?
- 기후 변화 대응에 있어 탄소세는 효과적인 수단인가?
```

**② AI 모델 선택**
- 카드를 클릭하여 참가할 AI를 선택/해제합니다.
- **최소 2개 이상** 선택 필요 (기본값: 3개 모두 선택)
- 목록은 DB에서 실시간으로 불러옵니다 (비활성 모델 자동 제외)

| AI | 모델명 | 특징 |
|----|--------|------|
| GPT-4o | gpt-4o | 최첨단 추론·코딩 |
| Claude Opus | claude-opus-4-8 | 심층 분석·글쓰기 |
| Gemini 2.5 | gemini-2.5-flash | 멀티모달·검색 |

**③ 라운드 수 선택**

| 라운드 | 특징 | 예상 시간 |
|--------|------|-----------|
| 1 Round | 빠른 비교 | 30~60초 |
| 2 Round | 기본 토론 | 1~2분 |
| 3 Round | 심층 토론 | 2~4분 |

**④ 토론 시작 버튼 클릭**
- 로딩 후 자동으로 토론 페이지로 이동합니다.

### 화면 4 — 토론 진행 페이지

접속 주소: `http://localhost/debate/{id}`

토론이 시작되면 실시간으로 각 AI의 답변이 표시됩니다.

#### 진행 단계 표시 (상단 Progress Bar)

```
[토론] ─────── [Judge 평가] ─────── [합의 도출]
```

#### 토론 타임라인

- **Round 1** — 초기 답변: 각 AI가 독립적으로 답변
- **Round 2** — 검토 & 반박: 상대 답변을 보고 보완·반박
- **Round 3** — 최종 입장: 모든 논의를 종합한 최종 결론

각 라운드 헤더 클릭으로 접기/펼치기 가능합니다.

#### 토론 완료 후 자동 표시

**Judge AI 평가 결과**
- 5개 항목별 점수 (0~10점)
- 승자 AI 강조 표시
- Judge 총평 텍스트

**최종 합의 결론**
- 신뢰도 점수 (0~100%)
- AI별 기여도 분포 바
- 종합 결론 Markdown 렌더링

> 기존에 완료된 토론을 다시 열면 DB에서 데이터를 불러와 표시합니다 (SSE 재연결 없음).

```json
  "plan": "free",
```

### 토론 API

#### 토론 시작 (인증 필요)

```http
POST /api/v1/debate/start
Authorization: Bearer {access_token}
Content-Type: application/json

{
  "topic": "2030년 가장 유망한 투자 자산은?",
  "selected_models": ["gpt", "claude", "gemini"],
  "num_rounds": 3
}
```

**응답**
```json
{
  "conversation_id": "uuid",
  "status": "pending"
}
```

> Authorization 헤더 없이 호출 시 `401 Unauthorized` 반환

---

#### 토론 목록 조회

```http
GET /api/v1/debate/
```

최근 50건 반환 (최신 순)

---

#### 토론 상세 조회

```http
GET /api/v1/debate/{conversation_id}
```

---

#### 실시간 토론 스트리밍 (SSE)

```http
GET /api/v1/debate/{conversation_id}/stream
Accept: text/event-stream
```

**이벤트 타입**

| type | 설명 | 주요 필드 |
|------|------|-----------|
| `connected` | 연결 완료 | conversation_id |
| `round_response` | AI 답변 도착 | round_no, provider, display_name, content |
| `round_complete` | 라운드 완료 | round_no |
| `judge_complete` | Judge 평가 완료 | winner, scores, summary |
| `consensus_complete` | 합의 완료 | final_answer, confidence_score |
| `done` | 전체 완료 | — |
| `error` | 오류 발생 | message |
| `timeout` | 응답 시간 초과 | — |

---

#### Judge 결과 조회

```http
GET /api/v1/judge/{conversation_id}
```

---

#### 합의 결과 조회

```http
GET /api/v1/consensus/{conversation_id}
```


### MAX_TOKENS_PER_RESPONSE 설정 가이드

| 값 | 한국어 분량 | 권장 용도 |
|----|-----------|----------|
| 1000 | 약 250~400자 | 간단한 비교 |
| 1500 | 약 400~600자 | 기본 토론 |
| **3000** | 약 800~1200자 | **권장 (심층 토론)** |
| 5000 | 약 1400~2000자 | 매우 상세한 답변 |

> Gemini는 max_output_tokens를 엄격한 상한으로 처리하여 토큰 소진 시 문장 중간에 잘릴 수 있습니다.
> GPT·Claude 대비 더 넉넉한 값을 설정하는 것이 좋습니다.

**해결**: 다른 웹 서버(IIS, Apache 등)가 실행 중일 수 있습니다.

```cmd
# 80번 포트 사용 프로세스 확인
netstat -ano | findstr :80

# docker-compose.yml에서 포트 변경 (예: 8080으로)
ports:
  - "8080:80"
```

### Q. AI 답변이 "[API key not configured]"로 표시됨

**해결**: `.env` 파일에 해당 AI의 API 키가 올바르게 입력되어 있는지 확인합니다.

```cmd
# 백엔드 컨테이너 재시작
docker-compose restart backend
```

---

### Q. Gemini 답변이 문장 중간에 잘림

**원인**: `MAX_TOKENS_PER_RESPONSE` 값이 낮아 출력이 강제 중단됩니다.

**해결**:
1. `.env` 파일에서 `MAX_TOKENS_PER_RESPONSE=3000` 으로 변경
2. 백엔드 서버 재시작 (`uvicorn` 또는 `docker-compose restart backend`)
3. 기존에 저장된 잘린 토론은 복구 불가 — 동일 주제로 새 토론 재실행 필요

### Q. Gemini 404 오류 — "This model is no longer available"

**원인**: `gemini-2.0-flash` 모델이 지원 종료됨.

**해결**: `.env` 파일에서 모델명 변경
```env
GEMINI_MODEL=gemini-2.5-flash
```
백엔드 재시작 후 적용.

### Q. 로그인 후 토론 시작 버튼이 여전히 "로그인하고 토론 시작"으로 표시됨

**원인**: 브라우저의 localStorage에 인증 상태가 저장되지 않은 경우.

**해결**:
1. 브라우저 개발자 도구 → Application → Local Storage → `arena_token` 키 확인
2. 없으면 로그아웃 후 재로그인
3. 여전히 문제 시 쿠키 + 캐시 전체 초기화 후 재시도

---

### Q. 로그인 상태가 새로고침 후 사라짐

**원인**: Zustand persist 설정 문제 또는 localStorage 접근 차단.

**해결**: 브라우저 설정에서 localStorage 허용 확인.
정상이라면 `arena_auth` 키에 토큰이 저장되어 새로고침 후에도 유지됩니다.

### Q. 토론이 시작 후 오랫동안 "연결 중..." 상태

**원인**: AI API 호출 지연, 네트워크 문제, 또는 개발 모드에서 React Strict Mode 이슈

**해결**:

2. API 키 유효성 및 크레딧 잔액 확인
3. `DEBATE_TIMEOUT_SECONDS` 값을 늘려봄 (기본 120초)
4. 개발 모드라면 uvicorn 재시작 후 새로운 토론 시작

## 9. 아키텍처 개요

```
사용자 브라우저
      │
      ▼
  Nginx (port 80)
  ├── /api/v1/* ──→ Backend (FastAPI, port 8000)
  └── /*         ──→ Frontend (Next.js, port 3000)

Frontend (Next.js + Zustand)
  ├── authStore   ← JWT 토큰 + 유저 정보 (localStorage persist)
  ├── debateStore ← 토론 진행 상태 관리
  └── axios client← Bearer 토큰 자동 첨부 (interceptor)

Backend (FastAPI)
  ├── Auth Layer  (JWT 발급 / 검증 / 회원 관리)
  ├── AI Adapters (OpenAI / Anthropic / Gemini)
  ├── Debate Engine   ← asyncio.gather 병렬 호출
  ├── Judge Engine    ← GPT-4o로 채점
  ├── Consensus Engine← 합의 도출
  ├── Ranking Engine  ← ELO 점수 계산
  └── PostgreSQL      ← 토론 데이터 영구 저장

실시간 통신
  Backend ──SSE──→ Browser (토론 진행 상황 스트리밍)

인프라 (Docker Compose)
  ├── postgres:16   ← 데이터 저장
  ├── redis:7       ← 캐시 (향후 Celery 큐)
  ├── backend       ← FastAPI 앱
  ├── frontend      ← Next.js 앱
  └── nginx         ← 리버스 프록시
```

### 인증 흐름

```
① POST /auth/register → DB 저장 → 자동 로그인
   또는
   POST /auth/login → 비밀번호 검증 → JWT 반환

② 브라우저 localStorage에 access_token 저장

③ 이후 모든 API 요청:
   axios interceptor → Authorization: Bearer {token} 자동 첨부

④ POST /debate/start 요청 시:
   Backend → JWT 검증 → 유효하면 토론 시작
              → 없거나 만료되면 401 반환
```

### 토론 실행 흐름

```
① POST /debate/start (+ Bearer token)
      │
      ▼
② Conversation 레코드 생성 (status: pending)
      │
      ▼
③ BackgroundTask 실행 (비동기)
      │
      ├─ Round 1: asyncio.gather(GPT, Claude, Gemini) — max_tokens: 3000
      ├─ Round 2: 이전 답변 컨텍스트 포함 재호출
      └─ Round 3: 최종 입장 요청
      │
      ▼
④ Judge AI 평가 (GPT-4o로 5항목 채점)
      │
      ▼
⑤ Consensus 생성 (공통 결론 도출)
      │
      ▼
⑥ ELO 랭킹 업데이트

⑦ (동시 진행) SSE Stream → 브라우저에 이벤트 push
⑧ 토론 재열람 시: REST API → DB에서 저장된 데이터 로드 (SSE 없음)
```
