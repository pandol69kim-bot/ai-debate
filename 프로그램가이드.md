## 1. 프로그램 소개

**AI Arena**는 여러 AI 모델(GPT-4o, Claude Opus, Gemini 2.0)에 동일한 질문을 던지고, AI들이 서로 토론·반박한 뒤 Judge AI가 최종 승자를 선정하는 플랫폼입니다.

### 핵심 기능

| 기능 | 설명 |
|------|------|
| **멀티 AI 동시 질의** | 선택한 AI들이 동일한 질문에 병렬로 답변 |
| **AI 토론 (최대 3라운드)** | 라운드마다 상대 답변을 보고 반박·보완 |
| **Judge AI 평가** | 정확성·논리성·근거·창의성·실현가능성 5항목 채점 |
| **합의 결론 도출** | AI들의 공통 의견을 종합한 최종 결론 생성 |
| **ELO 랭킹** | 토론 결과가 누적되어 AI 모델 순위 형성 |
| **실시간 스트리밍** | 토론 진행 상황을 실시간으로 화면에 출력 |

## 5. 화면별 사용법

### 화면 1 — 홈 (대시보드)

접속 주소: `http://localhost`

#### 토론 시작하기

**① 토론 주제 입력**
- 텍스트 박스에 토론할 주제나 질문을 입력합니다.
- 하단의 예시 버튼을 클릭하면 주제가 자동 입력됩니다.

```
예시 주제:
- 2030년 가장 유망한 투자 자산은 무엇인가?
- AI가 인간의 일자리를 대체할 것인가?
- 원격 근무와 사무실 근무 중 어느 것이 더 효율적인가?
```

**② AI 모델 선택**
- 카드를 클릭하여 참가할 AI를 선택/해제합니다.
- **최소 2개 이상** 선택 필요 (기본값: 3개 모두 선택)

| AI | 특징 |
|----|------|
| GPT-4o | 최첨단 추론·코딩 |
| Claude Opus | 심층 분석·글쓰기 |
| Gemini 2.0 | 멀티모달·검색 |

**③ 라운드 수 선택**

| 라운드 | 특징 | 예상 시간 |
|--------|------|-----------|
| 1 Round | 빠른 비교 | 30~60초 |
| 2 Round | 기본 토론 | 1~2분 |
| 3 Round | 심층 토론 | 2~4분 |

**④ 토론 시작 버튼 클릭**
- 로딩 후 자동으로 토론 페이지로 이동합니다.

---

### 화면 2 — 토론 진행 페이지

접속 주소: `http://localhost/debate/{id}`

토론이 시작되면 실시간으로 각 AI의 답변이 표시됩니다.

#### 진행 단계 표시 (상단 Progress Bar)

```
[토론] ─────── [Judge 평가] ─────── [합의 도출]
```

#### 토론 타임라인

- **Round 1** — 초기 답변: 각 AI가 독립적으로 답변
- **Round 2** — 검토 & 반박: 상대 답변을 보고 보완·반박
- **Round 3** — 최종 입장: 모든 논의를 종합한 최종 결론

각 라운드 헤더 클릭으로 접기/펼치기 가능합니다.

#### 토론 완료 후 자동 표시

**Judge AI 평가 결과**
- 5개 항목별 점수 (0~10점)
- 승자 AI 강조 표시
- Judge 총평 텍스트

**최종 합의 결론**
- 신뢰도 점수 (0~100%)
- AI별 기여도 분포 바
- 종합 결론 Markdown 렌더링

### 화면 3 — 랭킹 페이지

접속 주소: `http://localhost/rankings`

#### 기간 필터

| 탭 | 설명 |
|----|------|
| 전체 | 누적 전체 기간 ELO 순위 |
| 이번 달 | 당월 토론 기준 순위 |

## 6. API 레퍼런스

Swagger UI: `http://localhost/docs`

### 토론 시작

```http
POST /api/v1/debate/start
Content-Type: application/json

{
  "topic": "2030년 가장 유망한 투자 자산은?",
  "selected_models": ["gpt", "claude", "gemini"],
  "num_rounds": 3
}
```

**응답**
```json
{
  "conversation_id": "uuid",
  "status": "pending"
}
```

---

### 실시간 토론 스트리밍 (SSE)

```http
GET /api/v1/debate/{conversation_id}/stream
Accept: text/event-stream
```

**이벤트 타입**

| type | 설명 | 주요 필드 |
|------|------|-----------|
| `connected` | 연결 완료 | conversation_id |
| `round_response` | AI 답변 도착 | round_no, provider, content |
| `round_complete` | 라운드 완료 | round_no |
| `judge_complete` | Judge 평가 완료 | winner, scores, summary |
| `consensus_complete` | 합의 완료 | final_answer, confidence_score |
| `done` | 전체 완료 | — |
| `error` | 오류 발생 | message |

## 8. 트러블슈팅

### Q. `start.bat` 실행 시 "Docker가 실행 중이 아닙니다" 오류

**해결**: Docker Desktop을 먼저 실행하고 고래 아이콘이 초록색이 된 후 다시 시도합니다.

---

### Q. 포트 80이 이미 사용 중이라는 오류

**해결**: 다른 웹 서버(IIS, Apache 등)가 실행 중일 수 있습니다.

```cmd
# 80번 포트 사용 프로세스 확인
netstat -ano | findstr :80

# docker-compose.yml에서 포트 변경 (예: 8080으로)
ports:
  - "8080:80"
```

---

### Q. AI 답변이 "[API key not configured]"로 표시됨

**해결**: `.env` 파일에 해당 AI의 API 키가 올바르게 입력되어 있는지 확인합니다.

```cmd
# 백엔드 컨테이너 재시작
docker-compose restart backend
```

---

### Q. 토론이 시작 후 오랫동안 "연결 중..." 상태

**원인**: AI API 호출 지연 또는 네트워크 문제

**해결**:
1. `docker-compose logs backend` 로 오류 메시지 확인
2. API 키 유효성 및 크레딧 잔액 확인
3. `DEBATE_TIMEOUT_SECONDS` 값을 늘려봄 (기본 120초)

---

### Q. 랭킹 페이지에 데이터가 없음

**원인**: 토론을 완료해야 랭킹이 집계됩니다.

**해결**: 토론을 1회 이상 완료 (done 상태)하면 자동으로 랭킹이 생성됩니다.

---

### Q. 데이터베이스 초기화 (전체 데이터 삭제)

```cmd
cd "D:\_웹어플\AI 멀티 모델 경쟁·토론 플랫폼"
```

---

### Q. 백엔드 컨테이너가 계속 재시작됨

**확인**:
```cmd

```

주요 원인:

- Python 의존성 오류 → `requirements.txt` 확인

## 9. 아키텍처 개요

```
사용자 브라우저
      │
      ▼
  Nginx (port 80)
  ├── /api/v1/* ──→ Backend (FastAPI, port 8000)
  └── /*         ──→ Frontend (Next.js, port 3000)

Backend
  ├── AI Adapters (OpenAI / Anthropic / Gemini)
  ├── Debate Engine   ← asyncio.gather 병렬 호출
  ├── Judge Engine    ← GPT-4o로 채점
  ├── Consensus Engine← 합의 도출
  ├── Ranking Engine  ← ELO 점수 계산
  └── PostgreSQL      ← 토론 데이터 영구 저장

실시간 통신
  Backend ──SSE──→ Browser (토론 진행 상황 스트리밍)

인프라 (Docker Compose)
  ├── postgres:16   ← 데이터 저장
  ├── redis:7       ← 캐시 (향후 Celery 큐)
  ├── backend       ← FastAPI 앱
  ├── frontend      ← Next.js 앱
  └── nginx         ← 리버스 프록시
```

### 토론 실행 흐름

```
① POST /debate/start
      │
      ▼
② Conversation 레코드 생성 (status: pending)
      │
      ▼
③ BackgroundTask 실행 (비동기)
      │
      ├─ Round 1: asyncio.gather(GPT, Claude, Gemini)
      ├─ Round 2: 이전 답변 컨텍스트 포함 재호출
      └─ Round 3: 최종 입장 요청
      │
      ▼
④ Judge AI 평가 (GPT-4o로 5항목 채점)
      │
      ▼
⑤ Consensus 생성 (공통 결론 도출)
      │
      ▼
⑥ ELO 랭킹 업데이트

⑦ (동시 진행) SSE Stream → 브라우저에 이벤트 push
```

```python
class MistralAdapter(BaseAIAdapter):
    provider = "mistral"
    display_name = "Mistral Large"

    async def generate(self, messages, max_tokens=1000) -> AIResponse:
        # Mistral API 호출 구현
        ...

    async def stream(self, messages, max_tokens=1000):
        # 스트리밍 구현
        ...
```

**Step 2**: `backend/app/adapters/registry.py` 에 등록

```python
from app.adapters.mistral_adapter import MistralAdapter

_adapters = {
    "gpt": OpenAIAdapter(),
    "claude": AnthropicAdapter(),
    "gemini": GeminiAdapter(),

```
