## 1. Gemini 토론 내용 짤림 수정

### 증상

Gemini 2.5가 한국어로 답변을 생성할 때 문장 중간에서 잘려서 표시됨.
예: `...핵심적인 중견국(Middle Power)으로서의 위상을 더욱 공` 에서 종료.

### 원인 분석

| 원인 | 내용 |
|------|------|
| **낮은 max_tokens** | `MAX_TOKENS_PER_RESPONSE=1500` — 한국어는 영어 대비 토큰 효율이 낮아 금방 소진 |
| **불안정한 텍스트 추출** | `response.text` 직접 접근 시 안전 필터 차단·candidates 없음 등 예외 상황에서 `ValueError` 발생 또는 부분 텍스트만 반환 |
| **채팅 API 오버헤드** | `chat.start_chat()` → `send_message_async()` 방식은 히스토리 관리로 토큰 낭비 + 응답 동작 불일치 |
| **finish_reason 미처리** | `MAX_TOKENS`로 잘렸을 때 그대로 저장 |

### 수정 내역

#### `backend/app/core/config.py`

```python
# 변경 전
MAX_TOKENS_PER_RESPONSE: int = 1500
DEBATE_TIMEOUT_SECONDS: int = 120

# 변경 후
MAX_TOKENS_PER_RESPONSE: int = 3000
DEBATE_TIMEOUT_SECONDS: int = 180
```

#### `backend/app/adapters/gemini_adapter.py` — 전면 재작성

**변경 1 — 텍스트 추출 방식 개선**
```python
# 변경 전: response.text 직접 접근 (불안정)
full_text = response.text

# 변경 후: candidates[0].content.parts 순회로 명시적 추출
def _extract_text(response) -> str:
    parts = response.candidates[0].content.parts
    return "".join(p.text for p in parts if hasattr(p, "text") and p.text)
```

**변경 2 — API 호출 방식 변경**
```python
# 변경 전: chat API (히스토리 관리 오버헤드)
chat = model.start_chat(history=history)
response = await chat.send_message_async(prompt or "")

# 변경 후: generate_content_async 직접 호출 (더 안정적)
contents = _to_contents(messages)   # OpenAI 형식 → Gemini contents 변환
response = await model.generate_content_async(contents)
```

**변경 3 — MAX_TOKENS 이어쓰기**
```python
# finish_reason == MAX_TOKENS 감지 시 이어쓰기 1회 자동 실행
if _finish_reason(response) in _MAX_TOKENS_REASONS and full_text:
    cont_contents = contents + [
        {"role": "model", "parts": [{"text": full_text}]},
        {"role": "user",  "parts": [{"text": "이어서 계속 작성해주세요."}]},
    ]
    cont_response = await model.generate_content_async(cont_contents)
    full_text = full_text + _extract_text(cont_response)
```

**변경 4 — 안전 필터 완화**
```python
# 한국어 토론 텍스트가 안전 필터에 차단되는 상황 방지
_SAFETY_SETTINGS = {
    HarmCategory.HARM_CATEGORY_HARASSMENT: HarmBlockThreshold.BLOCK_NONE,
    HarmCategory.HARM_CATEGORY_HATE_SPEECH: HarmBlockThreshold.BLOCK_NONE,
    HarmCategory.HARM_CATEGORY_SEXUALLY_EXPLICIT: HarmBlockThreshold.BLOCK_MEDIUM_AND_ABOVE,
    HarmCategory.HARM_CATEGORY_DANGEROUS_CONTENT: HarmBlockThreshold.BLOCK_MEDIUM_AND_ABOVE,
}
```

## 2. API Key 상태 표시 기능 추가

### 내용

AI 모델 목록 API에서 각 모델의 API 키 설정 여부를 함께 반환하도록 수정.

### 수정 파일

#### `backend/app/schemas/schemas.py`

```python
class AIModelOut(BaseModel):
    id: UUID
    provider: str
    model_name: str
    display_name: str
    is_active: bool
    api_key_configured: bool = True   # ← 추가
```

#### `backend/app/api/routes/models.py`

```python
_API_KEY_MAP = {
    "gpt": settings.OPENAI_API_KEY,
    "claude": settings.ANTHROPIC_API_KEY,
    "gemini": settings.GOOGLE_API_KEY,
}

# list_models() 에서 api_key_configured 계산 후 반환
api_key_configured=bool(_API_KEY_MAP.get(m.provider, ""))
```

#### `frontend/src/types/index.ts`

```typescript
export interface AIModel {
  id: string;
  provider: string;
  model_name: string;
  display_name: string;
  is_active: boolean;
```
