## 핵심 기술 결정 사항

| 결정 | 이유 |
|------|------|
| `asyncio.gather`로 AI 병렬 호출 | 3개 AI 순차 호출 시 ~3배 느림 → 동시 호출로 응답 속도 최소화 |
| SSE (Server-Sent Events) 선택 | WebSocket 대비 단방향 스트리밍에 적합, nginx 설정 단순 |
| BackgroundTasks (FastAPI 내장) | Celery 없이도 MVP 기능 충분, 인프라 복잡도 감소 |
| 앱 시작 시 DB 자동 생성 + 시드 | 별도 마이그레이션 실행 없이 즉시 실행 가능 |
| ELO 알고리즘 (K=32) | 체스 표준 K값, 승패 이후 점수 수렴 속도 적절 |
| Next.js standalone output | Docker 이미지 크기 최소화, 운영 최적화 |

## API 엔드포인트 요약

```
POST   /api/v1/auth/register
POST   /api/v1/auth/login
POST   /api/v1/debate/start          ← 토론 시작, 백그라운드 실행
GET    /api/v1/debate/{id}/stream    ← SSE 실시간 이벤트
GET    /api/v1/debate/{id}           ← 토론 전체 조회
POST   /api/v1/models/query          ← 멀티 AI 동시 질의
GET    /api/v1/models/              ← 지원 모델 목록
GET    /api/v1/judge/{id}            ← Judge 결과
GET    /api/v1/consensus/{id}        ← 합의 결과
GET    /api/v1/rankings/?period=...  ← AI 랭킹
GET    /health                       ← 헬스체크
GET    /docs                         ← Swagger UI
```

## 실행 방법

### Docker 배포 (권장)
```bash
cp .env.example .env     # API 키 입력
start.bat                # 빌드 & 실행
# → http://localhost 접속
```

### 개발 모드
```bash
# 1. DB/Redis 실행
start-dev.bat

# 2. 백엔드
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000

# 3. 프론트엔드
cd frontend
npm install
npm run dev
# → http://localhost:3000
```
