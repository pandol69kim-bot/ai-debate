## 원인 분석

### 데이터 저장 위치

| 데이터 | 저장 위치 | 영구 보존 |
|--------|----------|---------|
| 토론 주제, 상태 | PostgreSQL `conversations` 테이블 | ✅ |
| 라운드별 AI 응답 | PostgreSQL `debate_rounds` 테이블 | ✅ |

→ **DB에는 모든 데이터가 영구 보존됨.** 문제는 프론트엔드가 DB를 직접 읽지 않는 것.

---

### 프론트엔드 구조 문제

**`frontend/src/app/debate/[id]/page.tsx`** 는 오직 SSE 스트림만 사용:

```
페이지 진입
  → SSE 연결 (GET /api/v1/debate/{id}/stream)
  → 실시간 이벤트 수신으로만 화면 구성
  → REST API (GET /api/v1/debate/{id}) 는 호출하지 않음
```

**Zustand 스토어**: `persist` 미들웨어 없음 → **페이지 새로고침 시 모든 상태 초기화**

---

### 시나리오별 동작

| 시나리오 | 결과 |
|---------|------|
| 토론 시작 후 같은 탭에서 결과 확인 | ✅ 정상 (SSE 실시간 수신) |
| 토론 중 새로고침 후 재접속 | ❌ 빈 화면 (store 초기화 + SSE 이벤트 재전송 없음) |
| 완료된 토론 URL 직접 접속 | ❌ `running` 상태로 멈춤 (5분 후 timeout) |
| uvicorn 재시작 후 토론 URL 접속 | ❌ 동일 |
| 과거 토론 목록 페이지 | ❌ 존재하지 않음 |

## 현재 가능한 확인 방법

### 방법 1. API 직접 호출 (개발자용)

백엔드 서버가 실행 중일 때 PowerShell에서:

```powershell
# 토론 전체 내용 (라운드 포함)
$id = "여기에-conversation-id"
Invoke-RestMethod "http://localhost:8000/api/v1/debate/$id" | ConvertTo-Json -Depth 10

# Judge 평가 결과
Invoke-RestMethod "http://localhost:8000/api/v1/judge/$id" | ConvertTo-Json -Depth 5

# 합의 결과
Invoke-RestMethod "http://localhost:8000/api/v1/consensus/$id" | ConvertTo-Json -Depth 5
```

### 방법 2. Swagger UI (브라우저)

```
http://localhost:8000/docs
→ GET /api/v1/debate/{conversation_id} → Try it out → conversation_id 입력
```

### 방법 3. DB 직접 조회

```powershell
docker exec -it <postgres_container> psql -U arena -d arena_db -c "
SELECT c.id, c.topic, c.status, c.created_at,
       count(dr.id) as round_count
FROM conversations c
LEFT JOIN debate_rounds dr ON c.id = dr.conversation_id
GROUP BY c.id ORDER BY c.created_at DESC LIMIT 10;"
```
