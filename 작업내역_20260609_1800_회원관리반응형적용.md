## 작업 내용

관리자 페이지(`/admin`) 의 **회원관리 탭(UsersTab)** 을 모바일 반응형으로 개선

### 수정 파일

| 파일 | 변경 내용 |
|------|----------|
| `frontend/src/app/admin/page.tsx` | `UsersTab` 컴포넌트 반응형 레이아웃 재작성 |

---

## 기존 문제

`UsersTab` 의 각 회원 행이 단일 `flex` 행으로 구성되어 있어
모바일(320px~480px)에서 다음 문제 발생:

- 권한배지 + 이름 + 이메일 + 토론수 + 가입일 + 버튼이 한 줄에 나열 → overflow
- 이메일이 긴 경우 버튼이 화면 밖으로 밀려남
- 터치 타깃 크기가 너무 작음 (`p-1.5`)

---

## 변경 사항

### 레이아웃 구조 변경

**변경 전** (단일 flex 행):
```
[관리자/일반] [이름(나) | 이메일·토론N건·가입일] [🛡️][🗑️]
```

**변경 후** (멀티 행 카드형):
```
[관리자/일반] [이름 (나)]              [🛡️][🗑️]
              [이메일@example.com]
              [토론 N건 · 2026. 1. 1. 가입]
```

### 세부 개선 사항

| 요소 | 기존 | 변경 |
|------|------|------|
| **이름 + (나)** | 한 줄 flex | `flex-wrap` — 좁으면 자동 줄바꿈 |
| **이메일** | 토론수·가입일과 같은 줄 | 별도 줄 분리, `truncate` 처리 |
| **토론수 · 가입일** | 이메일과 같은 행 | 별도 줄 분리, `flex-wrap` |
| **권한 배지** | `items-center` | `items-start mt-0.5` — 상단 정렬 |
| **액션 버튼** | `p-1.5` | `p-2` — 터치 영역 확대 |
| **헤더** | 제목만 표시 | 제목 + `N명` 회원수 카운트 추가 |

## 반응형 동작

| 화면 너비 | 동작 |
|----------|------|
| 320px (최소) | 이름/이메일/부가정보 3행, 버튼 우측 상단 고정 |
| 375px (모바일) | 동일, 이메일 truncate로 잘림 처리 |
| 640px (sm) | 여유 공간 생겨 자연스럽게 표시 |
| 768px (md) | 데스크톱 수준 가독성 확보 |
| 1024px+ | 넓은 여백으로 쾌적한 표시 |
